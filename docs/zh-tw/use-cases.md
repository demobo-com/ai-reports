# AI Agent 硬體應用案例

探討 AI agent 結合硬體的產品點子。每個點子都依據 [技術現況]({{ '/zh-tw/' | relative_url }}) 中的類別成熟度分數，標示 **可行性等級**：

- 🟢 **可落地** — 核心類別分數 ≥ 70；今年就能出貨，輔以標準流程防護。
- 🟠 **有挑戰** — 核心類別分數 40-69；MVP 可做，需容忍粗糙與窄領域。
- 🔴 **實驗性質** — 核心類別分數 < 40；demo 很酷，但對「自治」承諾要保守。

---

## 1. HDMI dongle：按需生成的數位看板 {% include mat-badge.html score=85 %}
{:#signage}

**主要類別：** CLI / 程式開發 (85) + 創意生成

一支 HDMI 棒接上任何電視，依自然語言指令即時生成數位看板——「今日特餐菜單」、「候診區歡迎字樣」、「3 號會議室，10 點全員會」。背後由 Claude Design 等結構化排版生成器驅動。

**為什麼簡單**
- 一次性生成——不需要長流程規劃，也沒有狀態要維護。
- 輸出可由人類視覺直接驗證。
- 任務範圍明確：矩形畫布、簡短文字、選用圖片。
- **硬體可以極度簡單——就是一台媒體播放器，從雲端儲存抓取預先由 AI 渲染好的素材（圖片 / 影片 / 動畫）播放。** 不需要瀏覽器、不需要即時渲染、不需要 HTML runtime。所有生成都在雲端完成，dongle 只負責播放排隊好的內容。

**顧客獲得的價值**
- **不用請設計師，也能有設計師等級的成品。** 非設計背景的使用者（店長、辦公室主管、活動主辦）也能做出看起來專業的版面。模型負責處理字型、層級、配色、留白。
- **不只有靜態畫面，還可以播動態內容。** 因為生成與顯示解耦，同一顆便宜的硬體也能播放任何雲端能渲染出來的東西——短影片循環、動態轉場、照片上的 Ken Burns 運鏡效果。
- **現場即時修正。** 站在電視前的員工可以直接說「標題放大」、「換成今天的湯」、「午餐菜單加上 QR code」。雲端重新生成素材、推送到 dongle，畫面幾秒內更新。不必來回聯繫外部設計師、不必重傳檔案、不必進 CMS 後台。
- **依情境自動換內容。** 綁定時間 / 日期 / 到場人數，看板就會自己換：午餐特餐到時間自動切成晚餐版、會議室自動更新「下一場」、候診區輪播當下最相關的訊息。
- **規模化成本低。** 每個變體只需生成一次，在 CDN 上快取。連鎖有 50 家分店，只付一次渲染費，不是 50 次。全網更新只是 cache 失效。

**輸入方式 — 使用者怎麼表達需求**

三種方案，各有硬體與體驗取捨：

1. **Dongle 上裝麥克風收語音** — 體驗最自然（「候診區歡迎字樣，標題放大」）。需要麥克風硬體與音訊處理管線——BOM 提高、隱私面增加。
2. **每台 dongle 綁定專屬 email 或 WhatsApp 帳號** — 使用者直接傳訊或寫信到綁定帳號（「今日特餐：芝麻葉沙拉」）。不必加麥克風，硬體維持簡單；文字明確、可追溯、非同步——多數情境已夠用。
3. **專屬網頁或 App** — 結構化表單，支援模板、媒體上傳、排程。控制最完整，但體驗最重——使用者必須打開 App。

**MVP 建議走方案 2（email / WhatsApp）。** 硬體成本最低、出貨最快，且符合中小商家員工日常溝通方式。語音可作為 v2 升級——等到出貨量支撐麥克風 BOM 再加。

**目標市場：** 餐廳、會議室、零售、中小辦公室、不動產開放看屋、診所、共享辦公大廳。

## 2. KVM 硬體：代替使用者操作電腦 / iPad / iPhone {% include mat-badge.html score=55 %}
{:#kvm}

**主要類別：** GUI / 電腦操作 (42)、試算表 (58)、溝通 / 助理 (38)

一個 USB-C 或硬體 KVM 橋接器，連接使用者的電腦、iPad 或 iPhone，讓遠端 agent 像人一樣操作——移動游標、輸入文字、點擊、擷取畫面。硬體本身便宜；**能力階梯** 才是可行性的關鍵。

| 任務 | 等級 | 為什麼 |
|---|---|---|
| iMessage / 簡訊自動回覆 | 🟢 可落地 | 已證實可行。短訊、已知收件人、例行內容。Agent 起草並送出。 |
| 自動下西洋棋 | 🟢 可落地 | 狀態空間有限；西洋棋引擎處理策略，agent 只需操作介面。 |
| 檔案整理（依內容重新命名、分類、加標籤） | 🟢 可落地 | 結構化、tool-use 類任務。Agent 讀檔名與中繼資料後執行。 |
| 批次處理 Excel 檔 | 🟠 有挑戰 | 試算表分數 58，乾淨資料可行，真實世界雜亂檔案會卡。 |
| 表單填寫（報稅、保險、員工入職） | 🟠 有挑戰 | Tool-use + GUI。標準表格可行；長篇多頁流程較難。 |
| 電腦疑難排解（磁碟清理、藍屏、病毒處理） | 🟠 有挑戰 | 跨管理工具的多步驟 GUI。對破壞性動作需要防護欄。 |
| 會議設定（加入 Zoom、分享螢幕、音訊控制） | 🟠 有挑戰 | Zoom / Meet / Teams 介面各異，動作空間小但邊界情況多。 |
| 軟體安裝與設定精靈 | 🟠 有挑戰 | 結構化但有分支的 GUI 流程。 |
| 修圖（Lightroom、Photoshop） | 🟠 有挑戰 | GUI 42。照食譜操作可以，品味與迭代不可靠。 |
| 3D 列印切片軟體 | 🔴 實驗 | GUI + 專業軟體。訓練資料稀少，錯誤會變成實體問題。 |
| 社群媒體內容與回覆 | 🔴 實驗 | 溝通 38。起草尚可，但對語氣、品牌聲音的判斷不可靠。 |
| 影片剪輯（DaVinci、Premiere） | 🔴 實驗 | GUI + 創意判斷 + 長流程。demo 有，生產級可靠性沒有。 |

**設計啟示：** KVM 以綠色等級任務為主要賣點（iMessage、西洋棋、檔案整理）。橘色等級定位為「需要人類監督」（疑難排解、表單、Excel、會議加入）。紅色等級定位為「預覽 / beta」，等分數爬上來。今年願意讓它處理 iMessage 的使用者，明年就會願意讓它修圖——階梯會往上填。

桌上常駐小裝置，以單一語音或聊天介面整合行事曆、email、Slack、智能家居 API。專屬硬體 agent 搭配良好的 tool-use 校準——隱私優先，工具集由使用者自行挑選。

- Tool-use 72 分，在明確、結構化的動作上表現穩健。

差異化：實體化 + 隱私 + 工具集策展。軟體已經 80% 到位；產品的 100% 在包裝。

## ~~5. 小型商家試算表助手~~ <span class="rejected-badge">未選用</span>
{:#spreadsheet-appliance}

**主要類別：** 試算表 / 檔案 (58)

桌上顯示器 + 麥克風，擺在商店 POS 或辦公桌旁。說一句「剛到 3 箱蘋果，同供應商」，就同步更新庫存、帳務與補貨點試算表。螢幕顯示即時摘要。

**為什麼有挑戰**
- 試算表 58 分在乾淨結構化資料上可行，但小商家的表格通常很雜亂。
- MVP 需要大量模板化——使用者先拿到預設 schema，而非自由輸入。

優勢：顯示器 + 語音形式繞開了 GUI 可靠性缺口（42 分）——後端直接改資料檔，不是驅動 Excel 或 Google Sheets 的介面。

## ~~7. 主管助理耳機~~ <span class="rejected-badge">未選用</span>
{:#earpiece}

**主要類別：** 溝通 / 助理 (38)

耳機 + 領夾式麥克風，全天聆聽使用者，低語提示（「5 分鐘後與 Alex 通話，上次他提到預算」）、起草 Slack / email 回覆、處理行事曆請求。對外動作一律需要人審。

**為什麼實驗**
- TheAgentCompany 自主任務完成率僅 30-39%。
- 對真實主管而言，全天自主助理尚未到可靠等級。
- 作為 **copilot**（建議但不送出）可行；自治 agent 不行。

誠實定位：這是「延伸記憶 + 建議引擎」，不是「你的 AI 參謀長」。起草協助真的好用；自治還沒到。

## 共通產品策略

1. **硬體要縮小問題，不要放大它。** HDMI 棒把任務限定為「生成看板」；通用平板則誘導使用者期待一切。限制才創造可靠性。

2. **審查關卡讓功能上線，而非讓自治上線。** 橘 / 紅等級的任務加一個「請審查」步驟就能用。同一個技術，自治時很糗，當 copilot 時很有用。

3. **2026 年，「隱私 + 在地運算」是一個真實的賣點。** 「你的金鑰、你的資料、看得見的實體硬體」這個敘事，對中小企業與企業內部尤其有效。

4. **讓強類別搭配弱類別。** 看板棒能成功，因為創意生成成熟、且任務被限定在一個矩形內。KVM 能力階梯之所以存在，是因為每個任務都在混合成熟類別（tool-use）與較弱類別（GUI）。針對弱點設計。

5. **主動 benchmark 分數每年爬 10-15 分。** 今天紅色的東西，18 個月後可能變橘色。蓋一個會隨著模型進步越來越好的產品，不是需要重做的產品。
