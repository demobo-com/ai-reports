# AI Agent 硬體應用案例

探討 AI agent 結合硬體的產品點子。每個點子都依據 [技術現況]({{ '/zh-tw/' | relative_url }}) 中的類別成熟度分數，標示 **可行性等級**：

- 🟢 **可落地** — 核心類別分數 ≥ 70；今年就能出貨，輔以標準流程防護。
- 🟠 **有挑戰** — 核心類別分數 40-69；MVP 可做，需容忍粗糙與窄領域。
- 🔴 **實驗性質** — 核心類別分數 < 40；demo 很酷，但對「自治」承諾要保守。

---

## 1. HDMI dongle：按需生成的數位看板 {% include mat-badge.html score=85 %}
{:#signage}

**主要類別：** CLI / 程式開發 (85) + 創意生成

一支 HDMI 棒接上任何電視，依自然語言指令即時生成數位看板——「今日特餐菜單」、「候診區歡迎字樣」、「3 號會議室，10 點全員會」。背後由 Claude Design 等結構化排版生成器驅動。

**為什麼簡單**
- 一次性生成——不需要長流程規劃，也沒有狀態要維護。
- 輸出可由人類視覺直接驗證。
- 任務範圍明確：矩形畫布、簡短文字、選用圖片。
- 硬體本身很簡單——就是一個瀏覽器，指向渲染伺服器。

**目標市場：** 餐廳、會議室、零售、中小辦公室、不動產開放看屋。

---

## 2. KVM 硬體：代替使用者操作電腦 / iPad / iPhone {% include mat-badge.html score=55 %}
{:#kvm}

**主要類別：** GUI / 電腦操作 (42)、試算表 (58)、溝通 / 助理 (38)

一個 USB-C 或硬體 KVM 橋接器，連接使用者的電腦、iPad 或 iPhone，讓遠端 agent 像人一樣操作——移動游標、輸入文字、點擊、擷取畫面。硬體本身便宜；**能力階梯** 才是可行性的關鍵。

| 任務 | 等級 | 為什麼 |
|---|---|---|
| 自動下西洋棋 | 🟢 可落地 | 狀態空間有限；西洋棋引擎處理策略，agent 只需操作介面。 |
| 批次處理 Excel 檔 | 🟠 有挑戰 | 試算表分數 58，乾淨資料可行，真實世界雜亂檔案會卡。 |
| 修圖（Lightroom、Photoshop） | 🟠 有挑戰 | GUI 操作 42，照食譜操作可以，品味與迭代仍吃力。 |
| 3D 列印切片軟體 | 🔴 實驗 | GUI + 專業軟體。訓練資料稀少，錯誤會變成實體問題。 |
| 社群媒體內容與回覆 | 🔴 實驗 | 溝通 38。起草尚可，但對語氣、脈絡、品牌聲音的判斷不可靠。 |
| 影片剪輯（DaVinci、Premiere） | 🔴 實驗 | GUI + 創意判斷 + 長流程。demo 有，生產級可靠性沒有。 |

**設計啟示：** 以西洋棋 / 試算表等級作為 KVM 的主要賣點。創意類任務定位為「預覽」或「beta」，而非成品級自治。今年願意把 Excel 交給它的使用者，明年會願意把照片交給它——因為分數在爬。

## 3. 程式碼自律工作站（過夜開發） {% include mat-badge.html score=85 %}
{:#coding-box}

**主要類別：** CLI / 程式開發 (85)

一個小型設備（Raspberry Pi 等級），連接開發者的 repo，配備限定 SSH + GitHub 憑證。下班前把 issue 丟給它，它過夜處理並產出 PR 等待早上審查。

**為什麼可行**
- SWE-bench Verified 頂尖模型達 87.6%——benchmark 衡量的程式碼工作已大致解決。
- 任務範圍天然受限（repo + issue tracker）。
- PR review gate 與人類協作模式完全一致。

硬體的差異化不在晶片——而在信任故事：**「設備只有你的金鑰、放在你家中、除了套件安裝外不會連對外網路。」**

---

## 4. 桌面 API 中樞 {% include mat-badge.html score=72 %}
{:#api-hub}

**主要類別：** API / 工具使用 (72)

桌上常駐小裝置，以單一語音或聊天介面整合行事曆、email、Slack、智能家居 API。專屬硬體 agent 搭配良好的 tool-use 校準——隱私優先，工具集由使用者自行挑選。

**為什麼可行**
- Tool-use 72 分，在明確、結構化的動作上表現穩健。
- 「幫我排 Jeff 的 1:1，30 分鐘，本週，下午為佳」正是目前 agent 最拿手的任務。

差異化：實體化 + 隱私 + 工具集策展。軟體已經 80% 到位；產品的 100% 在包裝。

---

## 5. 小型商家試算表助手 {% include mat-badge.html score=58 %}
{:#spreadsheet-appliance}

**主要類別：** 試算表 / 檔案 (58)

桌上顯示器 + 麥克風，擺在商店 POS 或辦公桌旁。說一句「剛到 3 箱蘋果，同供應商」，就同步更新庫存、帳務與補貨點試算表。螢幕顯示即時摘要。

**為什麼有挑戰**
- 試算表 58 分在乾淨結構化資料上可行，但小商家的表格通常很雜亂。
- MVP 需要大量模板化——使用者先拿到預設 schema，而非自由輸入。

優勢：顯示器 + 語音形式繞開了 GUI 可靠性缺口（42 分）——後端直接改資料檔，不是驅動 Excel 或 Google Sheets 的介面。

---

## 6. 深度網路研究設備 {% include mat-badge.html score=55 %}
{:#research-box}

**主要類別：** 瀏覽器 (55)、BrowseComp 類任務

桌上裝置，過夜跑持續搜尋任務——「調查 X 公司新產品線的所有公開資訊、摘要、附上來源」。基於 BrowseComp 類工作：答案在網路上，但需要耐心挖。

**為什麼有挑戰**
- 瀏覽器 55 分有用但脆弱。
- Anti-bot、登入牆、付費牆、動態 JS 都可能讓流程中斷。
- 開放或存檔來源表現較好，企業內部 dashboard 則卡關。

差異化：跑幾小時、需要追溯來源的任務，專屬長時 agent 遠優於「丟去問 ChatGPT」。

---

## 7. 主管助理耳機 {% include mat-badge.html score=38 %}
{:#earpiece}

**主要類別：** 溝通 / 助理 (38)

耳機 + 領夾式麥克風，全天聆聽使用者，低語提示（「5 分鐘後與 Alex 通話，上次他提到預算」）、起草 Slack / email 回覆、處理行事曆請求。對外動作一律需要人審。

**為什麼實驗**
- TheAgentCompany 自主任務完成率僅 30-39%。
- 對真實主管而言，全天自主助理尚未到可靠等級。
- 作為 **copilot**（建議但不送出）可行；自治 agent 不行。

誠實定位：這是「延伸記憶 + 建議引擎」，不是「你的 AI 參謀長」。起草協助真的好用；自治還沒到。

---

## 8. 窄領域家用任務機器人 {% include mat-badge.html score=22 %}
{:#robot}

**主要類別：** 裝置 / 機器人 (22)

只做一兩個明確家務的機器人——從籃子裡把衣服折好，或把餐具從預擺好的架上放進洗碗機。不是通用人形機器人。

**為什麼實驗**
- EmbodiedBench 最佳模型 28.9%；BEHAVIOR Challenge 冠軍進場低於 20%。
- 機器人是所有類別中最不成熟的，差距明顯。
- 任何宣稱更大範圍的產品，目前仍是科幻。

可行路線：在可標準化的物理環境中，做單一明確任務（鋪床、餵寵物、特定料理步驟）的窄領域機器人。

## 共通產品策略

1. **硬體要縮小問題，不要放大它。** HDMI 棒把任務限定為「生成看板」；通用平板則誘導使用者期待一切。限制才創造可靠性。

2. **審查關卡讓功能上線，而非讓自治上線。** 橘 / 紅等級的點子加一個「請審查」步驟就能用。同一個技術，自治時很糗，當 copilot 時很有用。

3. **2026 年，「隱私 + 在地運算」是一個真實的賣點。** 「你的金鑰、你的資料、看得見的實體硬體」這個敘事，對中小企業與企業內部尤其有效。

4. **讓強類別搭配弱類別。** 試算表助手能成立，正是因為用了有挑戰的試算表類別，但避開了更弱的 GUI 類別——後端改檔，不驅動介面。針對弱點設計。

5. **主動 benchmark 分數每年爬 10-15 分。** 今天紅色的東西，18 個月後可能變橘色。蓋一個會隨著模型進步越來越好的產品，不是需要重做的產品。
